#### 4. 文档归档策略

- 归档目录结构

- 归档操作流程

#### 5. 文档审查计划

```markdown

> **当前状态**: ⚠️ 过时 - [过时原因]  
> **最新报告**: 请参阅 [DOCUMENTATION_CODE_MATCH_AUDIT.md](DOCUMENTATION_CODE_MATCH_AUDIT.md) (2025-12-03)  
> **说明**: 本报告生成于[日期],[具体说明],仅供历史参考

---
```

## 6️⃣ 使用指南

### 6.1 如何查找文档?

**方法1: 通过文档中心**

1. 打开 `docs/README.md`
2. 查看分类目录
3. 点击文档链接

**方法2: 通过根目录README**

1. 打开 `README.md`
2. 找到"📚 文档资源"章节
3. 点击文档中心链接

**方法3: 快速搜索**

```bash
# 搜索关键词
grep -r "关键词" docs/

# 按文件名搜索
find docs/ -name "*关键词*.md"
```

### 6.2 如何判断文档是否最新?

**步骤1: 查看文档状态**

- 打开文档,查看开头的状态标记
- ✅ 最新/有效: 可放心使用
- ⚠️ 过时/历史: 仅供参考,查看最新文档链接

**步骤2: 查看最后更新日期**

- 每个文档都有"最后更新"字段
- 对比当前日期判断时效性

**步骤3: 交叉验证**

- 在docs/README.md中确认状态
- 查看是否有更新版本

### 6.3 如何更新文档?

**遵循文档生命周期管理规范**:

1. 阅读 `DOCUMENTATION_LIFECYCLE_STANDARD.md`
2. 确认需要更新的文档
3. 按照更新流程操作(4步)
4. 更新docs/README.md中的状态
5. 提交Git变更
