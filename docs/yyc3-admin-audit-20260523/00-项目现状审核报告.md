### 2.3 存在风险的问题 🔴

| # | 风险 | 位置 | 影响 | 缓解措施 |
| --- | --- | --- | --- | --- |
| 1 | **登录 API 使用硬编码 Mock 用户** | `app/api/auth/login/route.ts` | 生产环境安全风险极高，密码明文存储 | 接入真实数据库认证，密码使用 bcrypt 哈希 |
| 2 | **Token 生成使用 Base64 编码** | `app/api/auth/login/route.ts` `generateToken()` | 任何人可解码获取用户信息，无加密无签名 | 使用 `jose` 或 `jsonwebtoken` 库生成真实 JWT |
| 3 | **登录页面预填默认密码** | `app/page.tsx` `defaultValue="13103790379"` | 暴露测试账号信息 | 移除所有 defaultValue |
| 4 | **大量 `latest` 依赖标签** | `package.json` | 不同时间安装可能得到不同版本，构建不可复现 | 锁定所有版本号 |
| 5 | **CORS 配置 `Access-Control-Allow-Origin: *`** | `next.config.mjs` 登录 API | 允许任何来源的跨域请求 | 限制为具体域名 |
| 6 | **Redis 无认证配置** | `lib/cache/redis.ts` | 未配置 Redis 密码，生产环境风险 | 添加 `REDIS_PASSWORD` 环境变量 |
| 7 | **`k6` 作为 npm 依赖** | `package.json` dependencies | k6 是 Go 编写的二进制工具，npm 包已废弃/不可用 | 移除 npm 依赖，使用系统安装 k6 |
