### 2026-05-23 - Phase 1 安全修复执行

**操作类型**: 安全修复 / 功能改造
**涉及文件**:
- [lib/security/jwt.ts](../../lib/security/jwt.ts) ← 新建
- [lib/db/repositories/user-repository.ts](../../lib/db/repositories/user-repository.ts) ← 新建
- [app/api/auth/login/route.ts](../../app/api/auth/login/route.ts) ← 重写

- [app/page.tsx](../../app/page.tsx) ← 修改
- [next.config.mjs](../../next.config.mjs) ← 修改

**操作详情**:

**1. 安装安全依赖**:
- `pnpm add jose bcryptjs` — jose 用于 JWT（Edge Runtime 兼容），bcryptjs 用于密码哈希
- 发现警告: Next.js 15.5.4 有安全漏洞 (CVE-2025-66478)，需后续升级
- 发现警告: @vercel/kv 已废弃，需迁移到 Upstash Redis

**2. 创建 JWT 工具模块** `lib/security/jwt.ts`:
- 基于 jose 库，Edge Runtime 完全兼容
- HS256 签名算法，包含 issuer/audience 验证
- `generateToken()` — 生成包含 userId/role/storeId/permissions 的 JWT
- `verifyToken()` — 验证 Token 并返回解码后的 payload
- `getTokenRemainingTime()` — 获取 Token 剩余有效时间
- 默认 8 小时过期时间

**3. 创建用户仓库层** `lib/db/repositories/user-repository.ts`:
- `UserRepository` — 数据库用户查询（findByPhone/findById）
- `AuthService` — 统一认证服务，支持双模式:
  - **生产模式**: 数据库认证（bcrypt 密码验证）
  - **开发模式**: 数据库优先，数据库不可用时降级到 Mock 用户
- 保留 `13103790379` 作为开发预设测试号码
- 密码使用 bcryptjs 哈希验证

**4. 重写登录 API** `app/api/auth/login/route.ts`:
- 移除硬编码 MOCK_USERS 数据
- 使用 AuthService.authenticate() 进行认证
- 使用 generateToken() 生成真实 JWT Token
- 新增: 手机号格式验证（正则 `/^1[3-9]\d{9}$/`）
- 新增: IP 级登录频率限制（5次/15分钟）
- 修复: CORS 使用 `ALLOWED_ORIGINS` 环境变量替代通配符 `*`

**5. 重写认证管理 API** `app/api/auth/route.ts`:
- GET /auth/me: 使用 verifyToken() 验证 JWT，从数据库获取最新用户信息
- DELETE /auth/auth: 登出（客户端清除 Token 即可）
- 移除旧的 mockService 调用
- 移除 CORS 通配符 `*`

**6. 移除登录页默认凭据** `app/page.tsx`:
- 移除手机号输入框 `defaultValue="13103790379"`
- 移除密码输入框 `defaultValue="123456"`

**7. 修复 next.config.mjs**:
- 合并两个重复的 `experimental` 块为一个
- 添加 `localhost:5001` 到 serverActions.allowedOrigins
- CORS 从 `*` 改为 `ALLOWED_ORIGINS` 环境变量
- auth API 路由从 `/api/auth/login` 扩展为 `/api/auth/:path*`
- 恢复误删的 `images` 配置

**验证结果**:
