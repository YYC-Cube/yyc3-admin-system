# 📋 任务规划与节点目标

## 规划总则

> **执行有规划，规划有节点，节点有目标，目标可评估**

**本次会话核心目标**: 基于 YYC³ 团队规范，对项目进行全链路依赖升级和配置标准化，恢复可构建状态并确保代码质量门禁通过。

---

## 一、总体目标

1. **依赖体系标准化**: 将所有 `latest` 依赖固定为精确版本，统一使用 pnpm 包管理器
2. **配置文件规范化**: 合并双 next.config，修复 tsconfig 注释，统一端口配置
3. **构建验证通过**: 确保项目可正常安装依赖、通过类型检查、通过 lint 检查、成功构建
4. **安全基线加固**: 修复环境变量验证逻辑，收紧 CORS 策略，启用构建检查

---

## 二、阶段划分

| 阶段     | 名称               | 目标                                     | 交付物                      | 状态 |
| -------- | ------------------ | ---------------------------------------- | --------------------------- | ---- |
| Phase 1  | 依赖固定与包管理统一 | 所有依赖版本锁定，迁移到 pnpm            | 新 package.json + lockfile  | ⬜   |
| Phase 2  | 配置文件修复        | 合并 next.config，修复 tsconfig 等问题   | 修复后的配置文件            | ⬜   |
| Phase 3  | 依赖清理与优化      | 移除冗余依赖，分类调整 dev/prod          | 精简的 package.json         | ⬜   |
| Phase 4  | 代码修复与验证      | 修复环境变量验证、端口等逻辑问题         | 修复后的代码                | ⬜   |
| Phase 5  | 构建与质量验证      | install → lint → typecheck → build 全通过 | 构建成功报告                | ⬜   |

## 三、详细任务分解

### Task 1.1: 统一包管理器为 pnpm

**目标**: 将项目从 npm 迁移到 pnpm，建立确定性构建基础
**验收标准**:
- [ ] 创建 `pnpm-workspace.yaml`
- [ ] 更新 `.npmrc` 配置适配 pnpm
- [ ] package.json scripts 中的 `npm` 替换为 `pnpm`
- [ ] 执行 `pnpm install` 成功
- [ ] 生成 `pnpm-lock.yaml`
**依赖关系**: 无
**风险点**: 部分 npm 特有配置可能需要适配
**完成标记**: ⬜ 未开始

### Task 1.2: 固定所有 `latest` 依赖版本

**目标**: 将 ~60 个 `"latest"` 依赖全部替换为精确版本号
**验收标准**:
- [ ] 所有 `dependencies` 中的 `"latest"` 替换为具体版本
- [ ] 所有 `devDependencies` 中的范围版本确认合理性
- [ ] `pnpm install` 后 lockfile 记录精确版本
- [ ] 无依赖冲突或 peer dependency 警告
**依赖关系**: Task 1.1 完成后
**风险点**: 版本锁定可能暴露已存在的兼容性问题
**完成标记**: ⬜ 未开始

### Task 2.1: 合并双 next.config 文件

**目标**: 删除 `next.config.js`，将有效配置合并到 `next.config.mjs`
**验收标准**:
- [ ] 分析两个配置文件的所有有效配置项
- [ ] 合并所有配置到 `next.config.mjs`
- [ ] 修复 `experimental` 重复定义问题
- [ ] 删除 `next.config.js`
- [ ] 构建验证通过
**依赖关系**: 无
**风险点**: 遗漏 `next.config.js` 中的独有配置
**完成标记**: ⬜ 未开始

### Task 2.2: 修复 tsconfig.json

**目标**: 修复 tsconfig 中的 JSON 注释和过宽的 exclude
**验收标准**:
- [ ] 移除 `// </CHANGE>` 注释
- [ ] 确保所有必要的 `compilerOptions` 保留
- [ ] 调整 `exclude` 列表为合理范围
**依赖关系**: 无
**风险点**: 移除注释可能影响某些工具链
**完成标记**: ⬜ 未开始

### Task 2.3: 统一端口配置

**目标**: 所有配置文件中的端口统一为 5001
**验收标准**:
- [ ] `playwright.config.ts` 的 baseURL 和 webServer 端口改为 5001
- [ ] `next.config.mjs` 中 serverActions allowedOrigins 端口确认
- [ ] 文档中的端口引用统一
**依赖关系**: 无
**完成标记**: ⬜ 未开始

### Task 3.1: 依赖分类调整

**目标**: 将开发依赖移至 devDependencies，评估移除冗余依赖
**验收标准**:
- [ ] `@storybook/*`, `storybook` 移至 devDependencies
- [ ] 评估并移除 `k6`（非 npm 包）
- [ ] 评估并移除 `webpack`（Next.js 内置）
- [ ] 评估 `express`, `helmet`, `http-proxy-middleware` 使用情况
- [ ] `pnpm install` 无错误
**依赖关系**: Task 1.1
**完成标记**: ⬜ 未开始

### Task 4.1: 修复环境变量验证逻辑

**目标**: 修复 `env.validator.ts` 中的验证缺陷
**验收标准**:
- [ ] `validateEnv` 正确检查 `process.env[varName]` 而非 `!varName`
- [ ] 其他验证函数补充实际逻辑
- [ ] TypeScript 类型检查通过
**依赖关系**: 无
**完成标记**: ⬜ 未开始

### Task 4.2: 收紧 CORS 安全策略

**目标**: 将 CORS 配置从 `*` 改为环境变量控制的域名白名单
**验收标准**:
- [ ] `Access-Control-Allow-Origin` 使用环境变量
- [ ] 提供默认值和配置说明
- [ ] 不影响本地开发
**依赖关系**: 无
**完成标记**: ⬜ 未开始

### Task 4.3: 条件化构建检查禁用

**目标**: 仅在开发环境禁用 TS/ESLint 构建检查
**验收标准**:
- [ ] `ignoreBuildErrors` 仅在 `NODE_ENV !== 'production'` 时为 `true`
- [ ] `ignoreDuringBuilds` 同上
- [ ] 生产构建强制通过类型和 lint 检查
**依赖关系**: Task 2.1
**完成标记**: ⬜ 未开始

### Task 5.1: 全链路构建验证

**目标**: 验证所有修复后的完整构建流程
**验收标准**:
- [ ] `pnpm install` 成功
- [ ] `pnpm type-check` (tsc --noEmit) 通过
- [ ] `pnpm lint` 通过
- [ ] `pnpm build` 成功
- [ ] `pnpm dev` 启动正常
**依赖关系**: 所有前置任务
**完成标记**: ⬜ 未开始
